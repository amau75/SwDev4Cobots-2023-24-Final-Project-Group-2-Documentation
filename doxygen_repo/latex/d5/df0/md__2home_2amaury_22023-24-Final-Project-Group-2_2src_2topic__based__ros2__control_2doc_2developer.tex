pre-\/commit is a tool to automatically run formatting checks on each commit, which saves you from manually running clang-\/format (or, crucially, from forgetting to run them!).

Install pre-\/commit like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{pip3\ install\ pre-\/commit}

\end{DoxyCode}


Run this in the top directory of the repo to set up the git hooks\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{pre-\/commit\ install}

\end{DoxyCode}

 \hypertarget{md__2home_2amaury_22023-24-Final-Project-Group-2_2src_2topic__based__ros2__control_2doc_2developer_autotoc_md4}{}\doxysubsection{\texorpdfstring{Using ccache}{Using ccache}}\label{md__2home_2amaury_22023-24-Final-Project-Group-2_2src_2topic__based__ros2__control_2doc_2developer_autotoc_md4}
ccache is a useful tool to speed up compilation times with GCC or any other sufficiently similar compiler.

To install ccache on Linux\+: \begin{DoxyVerb}sudo apt-get install ccache
\end{DoxyVerb}
 For other OS, search the package manager or software store for ccache, or refer to the \href{https://ccache.dev/}{\texttt{ ccache website}}\hypertarget{md__2home_2amaury_22023-24-Final-Project-Group-2_2src_2topic__based__ros2__control_2doc_2developer_autotoc_md5}{}\doxysubsubsection{\texorpdfstring{Setup}{Setup}}\label{md__2home_2amaury_22023-24-Final-Project-Group-2_2src_2topic__based__ros2__control_2doc_2developer_autotoc_md5}
To use ccache after installing it there are two methods. you can add it to your PATH or you can configure it for more specific uses.

ccache must be in front of your regular compiler or it won\textquotesingle{}t be called. It is recommended that you add this line to your {\ttfamily .bashrc}\+: \begin{DoxyVerb}export PATH=/usr/lib/ccache:$PATH
\end{DoxyVerb}
 To configure ccache for more particular uses, set the CC and CXX environment variables before invoking make, cmake, catkin\+\_\+make or catkin build.

For more information visit the \href{https://ccache.dev/}{\texttt{ ccache website}}.
